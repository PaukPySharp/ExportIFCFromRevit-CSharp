# Папка `_settings`

В этой папке лежат конфигурационные файлы ExportIFC,
которые используются при запуске orchestrator.

В стандартном runtime-контуре `_settings` находится рядом с `admin_data`
и `ExportIfc.Orchestrator.exe`. В опубликованном orchestrator-пакете
такая раскладка уже готова. В репозитории по умолчанию `ExportIfc.Orchestrator.exe`
в корне не лежит, но при желании туда можно положить собранный или опубликованный `.exe`
и получить тот же рабочий контур. Именно такая раскладка делает запуск
предсказуемым, а диагностику — заметно проще.

## Что здесь лежит

### `settings.ini`

Это основной конфигурационный файл проекта. В нём задаются:

- базовые пути:
  - `dir_export_config` — корень каталога IFC-конфигов;
  - `dir_admin_data` — путь к рабочей папке `admin_data`;
- файловые настройки:
  - `config_json` — базовое имя основного JSON-конфига IFC;
- общие флаги работы:
  - `is_prod_mode` — использовать ли внешнюю `admin_data`
    по пути из `dir_admin_data`;
  - `run_revit` — выполнять ли реальный запуск Revit
    или ограничиться dry-run;
  - `enable_unmapped_export` — включать ли дополнительный
    no-map-маршрут с отдельным JSON из колонок E/F;
    при одновременном использовании mapped- и no-map-маршрутов
    каталоги выгрузки из колонок B и E должны различаться;
  - `revit_batch_timeout_minutes` — таймаут ожидания batch-пакета Revit;
- настройки Revit:
  - `revit_versions` — доступные версии запуска Revit;
  - `export_view3d_name` — имя 3D-вида для экспорта;
- имена листов Excel:
  - `sheet_path` — обязательный рабочий лист в `manage.xlsx`;
  - `sheet_ignore` — лист исключений в `manage.xlsx`;
  - `sheet_history` — лист истории в `history.xlsx`;
- структура IFC-конфигов:
  - `dir_common` — подпапка с общими JSON-конфигами;
  - `dir_layers` — подпапка с TXT-файлами сопоставления категорий Revit
    и классов IFC.

## Как это связано с остальными файлами

`settings.ini` нельзя рассматривать отдельно. Его нужно читать как минимум
в связке со следующими элементами:

- `admin_data/manage.xlsx` — здесь задаются проектные папки IFC-конфигов
  и имя файла из `dir_layers` для каждой рабочей строки;
- каталогом IFC-конфигов — здесь лежат общие JSON-конфиги,
  `Layer_Mapping.txt` и проектные `Export_Settings.json`;
- `admin_data/history/history.xlsx` — эта книга хранит рабочую
  историю состояний RVT-моделей и влияет на отбор задач
  в orchestrator.

Если лист `sheet_path` отсутствует, это считается фатальной ошибкой конфигурации.
Лист `sheet_ignore` при этом остаётся необязательным.

Важно не смешивать два разных уровня конфигурации:

- `Layer_Mapping.txt` — общий TXT-файл из папки `dir_layers`, который
  используется для сопоставления категорий Revit и классов IFC;
- `Property_Mapping.txt` — проектный файл, на который обычно
  ссылается `Export_Settings.json` внутри проектной папки.

Именно здесь чаще всего и появляется путаница,
если пытаться воспринимать всю IFC-конфигурацию как один плоский набор
файлов без уровней и ролей.

Отдельно важно понимать, что no-map-маршрут не отменяет общий
`Layer_Mapping.txt` из `dir_layers`. TXT-файл сопоставления категорий Revit
и классов IFC остаётся общим контрактом строки `manage.xlsx`,
а дополнительный маршрут меняет прежде всего JSON-конфигурацию,
которая задаётся через колонку F.

В штатном сценарии `settings.ini` лежит в `_settings`, поэтому при
`is_prod_mode = False` локальная `admin_data` вычисляется относительно
расположения `settings.ini` и в этом случае оказывается рядом
с корнем рабочей раскладки. Если же используется явный ini-файл
вне `_settings`, локальная `admin_data` вычисляется относительно
каталога этого ini-файла. Это не отменяет базовую рекомендацию:
основной запуск держать из корня рабочего контура, где рядом лежат
`ExportIfc.Orchestrator.exe`, `_settings` и `admin_data`.

## Кто обычно работает с этой папкой

Напрямую папку `_settings` обычно редактирует не рядовой моделировщик,
а тот, кто отвечает за настройку и сопровождение процесса:

- BIM-координатор или BIM-менеджер;
- технический администратор / BIM-администратор;
- разработчик ExportIFC.

Практически здесь обычно настраивают:

- пути к рабочим папкам;
- режим работы `admin_data`;
- список реальных версий Revit;
- имя экспортного 3D-вида;
- связку `_settings/settings.ini` ↔ `manage.xlsx` ↔ каталог IFC-конфигов.

> Важно: в стандартной раскладке файл `settings.ini` должен оставаться
> в папке `_settings`. Передача пути аргументом возможна,
> но не заменяет базовую дисциплину запуска из правильного рабочего корня.
