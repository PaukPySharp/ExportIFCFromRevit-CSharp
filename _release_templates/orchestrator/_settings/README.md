# Папка `_settings`

В этой папке лежит `settings.ini` для orchestrator-пакета.

В типовой release-раскладке `_settings` находится рядом с `admin_data`,
`run-local.cmd`, `_docs` и `ExportIfc.Orchestrator.exe`. Именно из такого корня
обычно выполняют first-run, dry-run и дальнейший ручной запуск пакета.

## Что здесь лежит

### `settings.ini`

Это основной конфигурационный файл orchestrator-пакета. В нём задаются:

- базовые пути:
  - `dir_export_config` — корень каталога IFC-конфигов;
  - `dir_admin_data` — путь к рабочей папке `admin_data`;
- файловые настройки:
  - `config_json` — базовое имя основного JSON-конфига IFC;
- общие флаги работы:
  - `is_prod_mode` — использовать ли внешнюю `admin_data`
    по пути из `dir_admin_data`;
  - `run_revit` — выполнять ли реальный запуск Revit
    или ограничиться dry-run;
  - `enable_unmapped_export` — включать ли дополнительный
    no-map-маршрут с отдельным JSON из колонок E/F;
    при одновременном использовании mapped- и no-map-маршрутов
    каталоги выгрузки из колонок B и E должны различаться;
  - `revit_batch_timeout_minutes` — таймаут ожидания batch-пакета Revit;
- настройки Revit:
  - `revit_versions` — доступные версии запуска Revit;
  - `export_view3d_name` — имя 3D-вида для экспорта;
- имена листов Excel:
  - `sheet_path` — обязательный рабочий лист в `manage.xlsx`;
  - `sheet_ignore` — лист исключений в `manage.xlsx`;
  - `sheet_history` — лист истории в `history.xlsx`;
- структура IFC-конфигов:
  - `dir_common` — подпапка с общими JSON-конфигами;
  - `dir_layers` — подпапка с TXT-файлами сопоставления категорий Revit
    и классов IFC.

## Что важно понимать в release-сценарии

`settings.ini` нужно читать как минимум в связке со следующими элементами:

- `admin_data/manage.xlsx` — здесь задаются проектные папки IFC-конфигов
  и имя файла из `dir_layers` для каждой строки;
- каталог IFC-конфигов — здесь лежат общие JSON-конфиги,
  `Layer_Mapping.txt` и проектные `Export_Settings.json`;
- `admin_data/history/history.xlsx` — эта книга хранит рабочую
  историю состояний RVT-моделей и влияет на отбор задач
  в orchestrator.

Если лист `sheet_path` отсутствует, это считается фатальной ошибкой конфигурации.
Лист `sheet_ignore` при этом остаётся необязательным.

`revit_versions` трактуется как список реально доступных запусков Revit.
Список может содержать пропуски между версиями: модель направляется в первую
доступную версию, которая не ниже версии самой модели.

Практический контракт такой:

- `Layer_Mapping.txt` — общий TXT-файл из папки `dir_layers`, который
  используется для сопоставления категорий Revit и классов IFC;
- `Property_Mapping.txt` — проектный файл, на который обычно
  ссылается `Export_Settings.json` внутри проектной папки.

Это тот минимум, который лучше понимать ещё до первого real-run:
так меньше шансов перепутать роли файлов и потом разбирать ошибки
в пакетном запуске.

Важно: no-map-маршрут не отменяет `Layer_Mapping.txt`.
TXT-файл сопоставления категорий Revit и классов IFC остаётся общим
контрактом строки `manage.xlsx`, а дополнительный маршрут меняет прежде
всего JSON-конфигурацию из колонки F.

Даже если путь к `settings.ini` передаётся явно, базовый release-сценарий
остаётся тем же: запускать пакет из рабочего корня, где рядом лежат
`ExportIfc.Orchestrator.exe`, `_settings` и `admin_data`. При
`is_prod_mode = False` локальная `admin_data` вычисляется относительно
расположения `settings.ini`; штатный случай этого пакета — ini внутри
`_settings`.

## Где смотреть подробности

Этот README — короткая памятка по папке `_settings` внутри release-пакета.

Для самого release-пакета сначала используйте `../README.txt`.
Если нужен более подробный, но всё ещё практический маршрут,
дополнительно откройте `../_docs/ExportIFC_manual_ru.docx`.
Полный manual находится в репозитории проекта `ExportIFCFromRevit-CSharp`
в файле `_docs/ExportIFC_manual.md`.

> Важно: в стандартной release-раскладке файл `settings.ini` должен
> оставаться в папке `_settings` рядом с `admin_data` и
> `ExportIfc.Orchestrator.exe`.
