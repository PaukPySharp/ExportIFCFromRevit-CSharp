# Пример файла с графиком запуска ExportIFC

Этот файл показывает, как можно отдельно зафиксировать фактическое
расписание запуска ExportIFC на конкретной машине или группе машин.

Речь идёт не о внутреннем формате проекта и не о конфигурационном файле,
который читает orchestrator. Это обычная рабочая памятка,
которую удобно держать в корне рабочего контура ExportIFC:
она нужна для того, чтобы в любой момент было видно,
когда именно Планировщик задач Windows должен запускать ExportIFC.

Подробная настройка запуска по расписанию описана в основном manual:

- [раздел 10.3. Запуск по расписанию через Планировщик задач Windows](./ExportIFC_manual.md#103-запуск-по-расписанию-через-планировщик-задач-windows);
- [раздел 12.2. Практические рекомендации по эксплуатации](./ExportIFC_manual.md#122-практические-рекомендации-по-эксплуатации).

README репозитория нужен как короткая входная точка,
а этот файл — как рабочая памятка по фактическому
графику запуска на вашей площадке.

## Пример ежедневного расписания для одного компьютера

```text
   Выгрузка IFC на ПК `IFC-SRV-01` запускается в:

   - 11:00
   - 16:00
   - 22:00
```

Имя компьютера можно подставить своё или не указывать,
если рабочая машина одна и путаницы не будет.

> Настройка расписания зависит от нагрузки на модели,
> инфраструктуру, окно обслуживания и внутренние регламенты.
> Этот файл — только пример, который нужно адаптировать
> под реальный режим вашей площадки.

## Как использовать этот пример

1. Создайте в корне рабочего контура ExportIFC
   текстовый файл, например `_График запуска программы.txt`.
2. Скопируйте в него этот пример и пропишите фактические времена запуска
   для ваших машин.
3. Если для разных машин используются разные окна запуска,
   зафиксируйте это явно в том же файле.
4. При изменении задачи в Планировщике задач Windows обновляйте и этот файл,
   чтобы локальная памятка не расходилась с реальным расписанием.
